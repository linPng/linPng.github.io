基础

@Override
[ˌəʊvəˈraɪd] vt. 推翻；不顾；践踏
表示重写.不写也可以,但是可以帮助检查和提示.

@Deprecated
[ˈdeprəkeɪtɪd] v. 不赞成；弃用；不宜用（deprecate的过去式及过去分词形式）
用于修饰已经过时的方法.

@SuppressWarnings
[səˈpres] vt. 抑制；镇压；废止 [ˈwɔːnɪŋ] n. 警告；预兆；预告 s
用于通知java编译器禁止特定的编译警告.

spring

@Component
[kəmˈpəʊnənt] n. 组成部分；成分；组件，元件
泛指组件，当组件不好归类的时候，我们可以使用这个注解进行标注

@Autowired
用来装配bean，都可以写在字段上，或者方法上。

默认情况下必须要求依赖对象必须存在，如果要允许null值，可以设置它的required属性为false，例如：@Autowired(required=false)

@Import
@Import只能用在类上，@Import通过快速导入的方式实现把实例加入spring的IOC容器中

@Transactional

[trænˈzækʃənəl]
声明式事务管理,只能在public方法上使用.基于 Spring 的动态代理的机制,实质是使用了 JDBC 的事务来进行事务控制的.

属性名 说明
name 当在配置文件中有多个 TransactionManager , 可以用该属性指定选择哪个事务管理器。
propagation 事务的传播行为，默认值为 REQUIRED。
isolation 事务的隔离度，默认值采用 DEFAULT。
timeout 事务的超时时间，默认值为-1。如果超过该时间限制但事务还没有完成，则自动回滚事务。
read-only 指定事务是否为只读事务，默认值为 false；为了忽略那些不需要事务的方法，比如读取数据，可以设置 read-only 为 true。
rollback-for 用于指定能够触发事务回滚的异常类型，如果有多个异常类型需要指定，各类型之间可以通过逗号分隔。
no-rollback- for 抛出 no-rollback-for 指定的异常类型，不回滚事务。

@PostConstruct
用来标记是在项目启动的时候执行这个方法。用来修饰一个非静态的void()方法

也就是spring容器启动时就执行，多用于一些全局配置、数据字典之类的加载

被@PostConstruct修饰的方法会在服务器加载Servlet的时候运行，并且只会被服务器执行一次。PostConstruct在构造函数之后执行,init()方法之前执行。PreDestroy（）方法在destroy()方法执行执行之后执

@EnableAsync
开启线程异步执行

@Async
异步方法

@Configuration
加载自定义配置类

@ConfigurationProperties
读取配置文件配置值

@Bean
声明配置类中的组件

@Order
定义Spring容器加载Bean的顺序

@ConditionalOnProperty
控制配置类是否生效

@ConditionalOnMissingBean
已存在就不注入bean

spring MVC

@Controller
标识一个该类是Spring MVC controller处理器,用来创建处理http请求的对象.基本被@RestController取代.

@RestController
Spring4之后加入的注解,原来在@Controller中返回json需要@ResponseBody来配合,如果直接用@RestController替代@Controller就不需要再配置@ResponseBody,默认返回json格式.

@Service
用于标注业务层组件,说白了就是加入你有一个用注解的方式把这个类注入到spring配置中

@RequestMapping
类定义处: 提供初步的请求映射信息,相对于 WEB 应用的根目录.

方法处: 提供进一步的细分映射信息,相对于类定义处的 URL.以下为限定请求类型的注解:

  • @GetMapping
  • @PostMapping
  • @PutMapping
  • @DeleteMapping

@RequestParam
用于将请求参数区数据映射到功能处理方法的参数上.

@Value
获取配置文件中的属性

Spring Boot

@SpringBootApplication
是Sprnig Boot项目的核心注解，目的是开启自动配置

@RestControllerAdvice
全局异常处理

@ExceptionHandler
定义一个异常的处理方法

@Mapper
在接口类上添加了@Mapper，在编译之后会生成相应的接口实现类

@MapperScan
是在Springboot启动类上面添加,指定要变成实现类的接口所在的包,然后包下面的所有接口在编译之后都会生成相应的实现类

@InitBinder
处理Controller的数据绑定,比如自动转化为Date类型.

spring cloud

@SpringCloudApplication
注解@SpringCloudApplication包括：@SpringBootApplication、@EnableDiscoveryClient、@EnableCircuitBreaker，分别是SpringBoot注解、注册服务中心Eureka注解、断路器注解。对于SpringCloud来说，这是每一微服务必须应有的三个注解，所以才推出了@SpringCloudApplication这一注解集合。

@EnableDiscoveryClient
实现服务注册与发现

@EnableRyFeignClients
启用feign客户端

@ComponentScan
自动扫描并加载符合条件的组件(比如@Component和@Repository等)或者bean定义,最终将这些bean定义加载到IoC容器中.

@LoadBalanced
RestTemplate在请求时拥有客户端负载均衡的能力

@EnableFeignClients
启用feign客户端

@FeignClient
定义feign客户端

spring security

@EnableWebSecurity
​ Spring Security用于启用Web安全的注解

@EnableGlobalMethodSecurity

  • @EnableGlobalMethodSecurity(securedEnabled=true)
    开启@Secured 注解过滤权限
  • @EnableGlobalMethodSecurity(jsr250Enabled=true)
    开启@RolesAllowed 注解过滤权限
  • @EnableGlobalMethodSecurity(prePostEnabled=true)
    使用表达式时间方法级别的安全性 4个注解可用
    -@PreAuthorize 在方法调用之前,基于表达式的计算结果来限制对方法的访问
    -@PostAuthorize 允许方法调用,但是如果表达式计算结果为false,将抛出一个安全性异常
    -@PostFilter 允许方法调用,但必须按照表达式来过滤方法的结果
    -@PreFilter 允许方法调用,但必须在进入方法之前过滤输入值

@EnableAuthorizationServer
启用授权服务器

@EnableResourceServer
启用oauth2服务

元注解

可以自定义注解,配合切面的@Pointcut和@annotation

@interface
定义新的Annotation类型使用@interface关键字(在原有interface关键字前增加@符号).允许定义了成员变量.

@Retention
用于指定Annotation可以保留多长时间.包含一个名为”value”的成员变量,该value成员变量是RetentionPolicy枚举类型.

  • RetentionPolicy.SOURCE：Annotation只保留在源代码中，编译器编译时，直接丢弃这种Annotation。
  • RetentionPolicy.CLASS：编译器把Annotation记录在class文件中。当运行Java程序时，JVM中不再保留该Annotation。
  • RetentionPolicy.RUNTIME：编译器把Annotation记录在class文件中。当运行Java程序时，JVM会保留该Annotation，程序可以通过反射获取该Annotation的信息。

@Target
指定Annotation用于修饰哪些程序元素.包含一个名为”value”的成员变量,该value成员变量类型为ElementType[ ],ElementType为枚举类型,值有如下几个:

  • ElementType.TYPE：能修饰类、接口或枚举类型
  • ElementType.FIELD：能修饰成员变量
  • ElementType.METHOD：能修饰方法
  • ElementType.PARAMETER：能修饰参数
  • ElementType.CONSTRUCTOR：能修饰构造器
  • ElementType.LOCAL_VARIABLE：能修饰局部变量
  • ElementType.ANNOTATION_TYPE：能修饰注解
  • ElementType.PACKAGE：能修饰包

@Documented
用javadoc命令生成API文档后,所有使用注解修饰的程序元素,将会包含注解的说明.

@Inherited
表示注释类型被自动继承.

表单校验注解

@Valid
用于验证注解是否符合要求

@Validated
是@Valid 的一次封装,是Spring提供的校验机制使用.而外支持分组.

@NotBlank
只能作用在String上，不能为null，而且调用trim()后，长度必须大于0

@NotNull
不能为null.但可以为empty.

@NotEmpty
不能为null.而且长度必须大于0.

@Size
校验基础类型的长度,一般用于String类型,成员变量:min,max控制长度的上下限.

@Email
校验邮箱格式.

不常使用的注解

Spring Cache

@Cacheable
注解在方法上,表示该方法的返回结果是可以缓存的.value(必填)属性指定Cache名称,key属性自定义key,condition属性指定发生的条件.

@CachePut
与@Cacheable不同的是使用@CachePut标注的方法在执行前不会去检查缓存中是否存在之前执行过的结果，而是每次都会执行该方法，并将执行结果以键值对的形式存入指定的缓存中。

@CacheEvict
用来标注在需要清除缓存元素的方法或类上的。

@ApiOperation

@ApiResponses

@ApilmplicitParams

@ApiResponse

@ApiModel

@ApiModelProperty

AOP

@Pointcut
