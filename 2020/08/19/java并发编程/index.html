并发编程的目的是让程序运行的更快,同时也带来了更多的挑战,比如上下文切换,死锁,资源问题.

并发编程的挑战

并不是线程越多越快,线程的创建和上下文的切换都要耗时.

上下文切换:任务从保存到再加载的过程.

  • 如何减少上下文切换
    • 无锁并发编程:通过id将数据分段处理,避免使用锁.
    • CAS算法
    • 使用最少线程
    • 协程:在单线程里实现多任务调度

死锁:两线程相互等待对方释放锁.

资源限制:最好要实现cpu,数据库读写,宽带速度的均匀使用,避免线程阻塞.

底层实现原理

volatile

volatile是轻量级的synchronize.是为了确保共享变量能被准确和一致的更新,所有线程看到的这个变量都是一致的.

实现原则:

  1. Lock前缀指令会使处理器缓存回写到内存
  2. 一个处理器的缓存回写到内存会使其他处理器的缓存失效

synchronize

可以对实例对象,class对象,代码块加锁.

java对象头:java对象头的markword存储了锁信息.

锁的升级与竞争

java1.6中,锁一共有4种:无锁状态,偏向锁状态,轻量锁状态,重量锁状态.锁会随着竞争情况逐渐升级,但是不能降级.

偏向锁:偏向锁不会主动释放锁,当线程获得偏向锁时会在对象头和栈帧中记录下现成id,在出现锁竞争才释放.

加锁和解锁不需要而外的消耗,和执行非同步方法相比仅存在纳秒级的差距;如果线程间存在竞争,会带来额外的锁撤销消耗;适用于只有一个线程访问同步块的场景.

偏向锁的升级:在出现两个线程持续的锁竞争时升级.

轻量锁:竞争的线程不会阻塞,提高了程序的响应速度;如果始终得不到锁竞争的线程,自旋会消耗cpu;适用于追求响应时间,同步块执行速度快的场景.

轻量锁的升级:自旋达到一定次数.

重量锁:线程竞争不使用自旋,但是会线程阻塞,适用于吞吐量大,执行较长的场景.

JAVA内存模型

线程通信

  • 共享内存 的并发模型：通过 读写内存中的公共状态 来进行隐式通信。
  • 消息传递 的并发模型：没有公共状态，只能 通过发送消息来显示的进行通信。

线程同步

  • 共享内存 的并发模型：同步时显示进行的。我们必须显示指定某段代码需要在线程直线互斥执行。
  • 消息传递 的并发模型：由于消息发送必须在消息接收之前，因此同步时隐式的。

Java并发 采用的是 共享内存模型，Java线程之前的通信总是隐式进行的。

Java内存模型的抽象结构

在Java中，所有 实例域、静态域 和 数组元素 都储存在堆内存中，堆内存在线程之前共享。
本文用 共享变量 统一描述 实例域、静态域 和 数组元素 。

局部变量 、方法定义参数、异常处理器参数 不会在内存之间共享，他们不会有内存可见性问题，也不受内存模型影响。

从抽象角度看，JMM定义了 线程 和 主内存 之间的抽象关系：线程之间的共享变量储存在主内存中，每个线程都有一个私有的本地内存，本地内存储存了 该线程 以读写共享变量的副本。

volatile的内存语义

volatile写的内存语义：当写一个volatile变量时，JMM 会把该线程对应的本地内存中的共享变量值刷新到主内存。
volatile读的内存语义：当读一个volatile变量时，JMM 会把该线程对应的本地内存置为无效。线程接下来将从主内存中读取共享变量。

锁的内存语义

当线程释放锁时，JMM会把该线程对应的本地内存中的共享变量刷新到主内存中。和 volatile 写 类似。
当线程获取锁时，JMM会把该线程对应的本地内存置为无效。和 volatile 读 类似。

线程

线程是操作系统能够进行运算调度的最小单位。一条线程指的是进程中一个单一顺序的控制流，一个进程中可以并发多个线程，每条线程并行执行不同的任务。

线程的状态

  • NEW 初始状态
  • RUNNABLE 运行状态
  • BLOCKED 阻塞状态
  • WAITING 等待状态
  • TIME_WAITING 超时等待状态
  • TERMINATED 终止状态

锁

  • Lock接口:在Java SE 5之后，并发包中新增了Lock接口（以及相关实现类）用来实现锁功能，它提供了与synchronized关键字类似的同步功能，只是在使用时需要 显式 地获取和释放锁。
    虽然它缺少了（通过synchronized块或者方法所提供的）隐式获取释放锁的便捷性，但是却拥有了锁获取与释放的 可操作性、可中断的获取锁 以及 超时获取锁 等多种synchronized关键字所不具备的同步特性。
  • 重入锁ReentrantLock:广义上的可重入锁指的是可重复可递归调用的锁，在外层使用锁之后，在内层仍然可以使用，并且不发生死锁（前提得是同一个对象或者class），这样的锁就叫做可重入锁。ReentrantLock和synchronized都是可重入锁.
  • 读写锁:一般情况下，读写锁 的性能都会比 排它锁 好，因为大多数场景 读是多于写 的。
    • 写锁是一个支持重进入的排它锁.
    • 如果存在读锁，则写锁不能被获取.
    • 读锁是一个支持重进入的共享锁.
    • 锁降级指的是写锁降级成为读锁.

并发容器与框架

  • ConcurrentHashMap

    • 在并发编程使用HashMap会导致死循环:在数组扩容将旧数组下的列表转移到新数组时,多线程操作或产生环形链导致死循环.
    • 线程安全的HashTable效率非常低.HashTable容器使用synchronized来保证线程安全，但在线程竞争激烈的情况下HashTable的效率非常低下。因为当一个线程访问HashTable的同步方法，其他线程也访问HashTable的同步方法时，会进入阻塞或轮询状态。如线程1使用put进行元素添加，线程2不但不能使用put方法添加元素，也不能使用get方法来获取元素，所以竞争越激烈效率越低。
    • ConcurrentHashMap的锁分段技术可有效提升并发访问率.以put操作为例,CAS方式确定key的数组下标,synchronized保证链表节点的同步效果.

  • ConcureentLinkedQueue:一个基于链接节点的无界线程安全队列.

  • Java中的阻塞队列

    • 支持阻塞的插入方法:当队列满时,队列会阻塞插入元素的线程,直到队列不满.

    • 支持阻塞的移除方法:在队列为空时,获取元素的线程会等待队列变为非空.

    • ArrayBlockingQueue:一个由数组结构组成的有界阻塞队列.

    • LinkedBlockingQueue:一个由链表结构组成的有界阻塞队列.

    • PriorityBlockingQueue:一个支持优先级排序的无界 阻塞队列.

    • DelayQueue:一个使用优先级队列实现的无界阻塞队列.

    • SynchronousQueue:一个不存储元素的阻塞队列.

    • LinkedTransferQueue:一个由链表结构组成的无界阻塞队列.

    • LinkedBlockingDeque:一个由链表结构组成的双向阻塞队列.

  • Fork / Join 框架:分治编程,先分割任务,然后执行任务并合并结果.

原子操作类

官方介绍

而 Java 从 JDK 1.5 开始提供了 java.util.concurrent.atomic 包（以下简称Atomic包），这个包中的 原子操作类 提供了一种用法简单、性能高效、线程安全地更新一个变量的方式。

因为变量的类型有很多种，所以在 Atomic 包里一共提供了 12个 类，属于以下 4 种类型的原子更新方式：

  • 原子更新基本类型
    • AtomicBoolean：原子更新布尔类型。
    • AtomicInteger：原子更新整型。
    • AtomicLong：原子更新长整型。
  • 原子更新数组
    • AtomicIntegerArray：原子更新整型数组里的元素。
    • AtomicLongArray：原子更新长整型数组里的元素。
    • AtomicReferenceArray：原子更新引用类型数组里的元素。
  • 原子更新引用
    • AtomicReference：原子更新对象引用。
    • AtomicMarkableReference：原子更新带有标记位的对象引用。
    • AtomicStampedReference：原子更新带有版本号的对象引用。
  • 原子更新属性（字段）
    • AtomicIntegerFieldUpdater：原子更新volatile修饰的整型的字段的更新器。
    • AtomicLongFieldUpdater：原子更新volatile修饰的长整型字段的更新器。
    • AtomicReferenceFieldUpdater：原子更新volatile修饰的引用类型里的字段的更新器。

并发工具类

JDK提供了几个并发工具类

  • CountDownLatch:等待多线程完成
  • CyclicBarrier:同步屏障
  • Semaphore:控制并发线程数
  • Exchanger:线程间交换数据

线程池

线程池的好处:

  1. 降低资源消耗.重复利用线程,减少创建和销毁线程的消耗.
  2. 提高响应速度.不必等待创建线程.
  3. 提高线程的可管理性.

线程池的处理流程(ThreadPoolExecutor execute):

  1. 线程池判断核心线程池是否已满?否:创建线程|是:下一流程
  2. 队列是否已满?否:储存在队列里|是:下一流程
  3. 线程池是否已满?否:创建线程|是:按照饱和策略处理任务.

合理配置

  • CPU密集型:配置较少的线程数,如:Runtime.getRuntime().availableProcessors()+1
  • IO密集型:配置较多的线程数,如Runtime.getRuntime().availableProcessors()*2
  • 混合型:可以考虑拆分成CPU密集型和IO密集型

创建线程

参考

corePoolSize:核心线程池大小,当任务到达线程池时,如果线程数小于核心线程数,不论是否有空闲线程都会创建线程.

maximumPoolSize:最大线程数,如果现有线程队列满了,且线程池未满就创建线程.如果设置了无界队列,则这个配置无效.

keepAliveTime:线程活动保持时间,线程在空闲后保持存活的时间,如果任务多且快,会频繁超过核心数,可以调大提高利用率.

TimeUnit:时间单位,常用TimeUnit.SECONDS.

workQueue:线程等待队列:

  • ArrayBlockingQueue:基于数组结构的有界阻塞队列.
  • LinkedBlockingQueue:基于链表的阻塞队列.
  • SynchronousQueue:不储存元素的阻塞队列,每次插入必须等到移除操作;这是一个内部没有任何容量的阻塞队列;
  • PriorityBlockingQueue具有优先级的无限阻塞队列

threadFactory:线程创建工厂,可以重写newThread方法,或者使用Executors.defaultThreadFactory();

handler:饱和策略:默认defaultHandler为AbortPolicy.

  • AbortPolicy:直接抛出异常.抛出java.util.concurrent.RejectedExecutionException异常.
  • CallerRunsPolicy:只用调用者所在的线程来运行任务.重试添加当前的任务,会自动重复调用execute()方法.
  • DiscardOldestPolicy:丢弃队列里最近的一个任务,执行当前任务.
  • DiscardPolicy:不处理,直接丢弃.
提交任务

有execute()和submit()两种.

execute():没有返回值,入参是Runnable类.

submit():返回Future对象通过get方法获取返回值,get方法会阻塞当前线程直到任务完成;使用get(long timeout,TimeUnit unit)方法则会阻塞当前线程一段时间后立即返回,任务不一定完成.入参有Callable\Runnable\Runnable,T 三种.

关闭线程池

可以通过shutdown或shutdownNow来关闭线程池.原理是通过遍历线程池中的工作线程,然后逐个调用线程的interrupt方法来中断线程,所以无法响应的线程可能永远无法终止.shutdownNow会停止所有正在执行的线程,shutdown会继续执行未完成的任务,通常用shutdown来关闭线程池.调用后isShutdown就返回true,关闭成功后isTerminaed返回true.

调用线程池的方法会被多次调用,多次调用假如要是有初始线程数2个,那每次调用都会有两个线程数空闲,为了不浪费资源,所以要shutdown.而声明在类里面,在Spring的bean声明中每个bean默认是单例的,它起一次在Spring项目启动的时候创建一个对象,假设对象有核心线程数2个,对象用完也就有2个线程空闲,不shutdown也无所谓.

Executor框架

从JDK5开始,工作单元和执行机制分离,工作单元包括:Runnable和Callable,执行机制:Executor(ExecutorService).

主要使用ThreadPoolExecutor,不推荐使用FixedThreadPool,SingleThreadExecutor,CachedThreadPool

  • Executors:提供了一系列工厂方法用于创建线程池,返回的线程池都实现了ExecutorService接口.
    • newFiexedThreadPool:创建一个定长线程池,可控制线程最大并发数,超出的线程会在队列中等待.
    • newCachedThreadPool:创建一个可缓存线程池,如果线程池长度超过处理需要,可灵活回收空闲线程,若无可回收,则新建线程.
    • SingleThreadExecutor:创建一个单线程化的线程池,它只会用唯一的工作线程来执行任务,保证所有任务按照指定顺序(FIFO, LIFO, 优先级)执行.
    • newScheduledThreadPool:支持定时以及周期性执行任务
  • Executor:一般用来代替Thread类的创建,只有execute方法.
  • ExecutorService:继承了Executor接口，且更加被广泛使用,增加了submit,shutdown等方法.
  • Future/FutureTask:用于表示异步计算的结果.
  • Runnable/Callable:用于实现线程要执行的工作单元.

实践

生产者和消费者模式就非常适合多线程,线程池就是实现方式.

在实践中,top命令可以帮助分析性能,command是java的,cpu是总和,可以按1查看多核的数据(us:用户空间,sy内核空间,ni:用户进程空间内改变过优先级的进程占用空间,id:空闲,wa:等待).
