Java中的实体类

JavaBean

JavaBean 通过提供符合一致性设计模式的公共方法将内部域暴露成员属性,set和get方法获取。类必须是具体的和公共的,并且具有无参数的构造器。类中访问属性的方法都必须是public的,一般属性是private的。

实体类包名/后缀的区别

以entity, domain ,model命名的包 , 包中的类都满足javaBean的规范

entity/PO/DO

entity包下的类中的所有属性与数据库表格中的字段名逐一对应.

PO (persistant object持久化对象)就是对应数据库中某个表中的一条记录,多个记录可以用PO的集合,PO中应该不包含任何对数据库的操作.

DO(Data Object)与数据库表结构一一对应,通过DAO层向上传输数据源对象.

domain/AO/POJO

domain包下的类中的属性除了包含数据库表格中的字段, 还可以包括其他属性

AO(Application Object)应用对象. 在Web层与Service层之间抽象的复用对象模型,极为贴近展示层,复用度不高.

Query:数据查询对象,各层接收上层的查询请求. 注意超过2个参数的查询封装，禁止使用Map类来传输。

POJO(Plain Old Java Objects,普通 Java 对象) 包下的类,可以是domain,可以是vo,也可以是entity;POJO实质上可以理解为简单的实体类.POJO主要用于数据的临时传递,它只能装载数据,作为数据存储的载体,而不具有业务逻辑处理的能力.在本阿里手册中,POJO专指只有setter/getter/toString的简单类,包括DO/DTO/BO/VO等,一般禁止命名.

model/VO

model/vo(view object)包下的类中的属性是服务器与页面交互的视图数据 , vo类中的属性与数据库表格数据没有关系;xxxVO,xxx一般为网页名称.

DTO

DTO （Data Transfer Object）数据传输对象,是接口之间传递的数据封装.

其他类

DAO(data access object)：数据访问对象

BO(business object):业务对象,由Service层输出的封装业务逻辑的对象.
